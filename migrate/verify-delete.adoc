= 验证并删除 FLI LUN 导入关系
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
导入副本完成且导入状态完成后，您可以通过删除与 lun import delete 的导入配对来完成迁移，这会在源 LUN 不再使用且目标 LUN 现在是主机 I/O 的活动副本后安全地删除导入关系。在脱机 FLI 工作流中，您通常会在导入和任何验证后使目标 LUN 联机，以便主机可以重新发现并恢复对迁移的 LUN 的访问。

如果在删除关系之前需要进行数据完整性确认，则可以运行 lun import verify，该验证会执行逐块比较，但会造成中断，因为验证必须脱机执行（这意味着 LUN 必须脱机，并且在验证期间主机无法访问）。迁移后操作（如主机补救、分区和 iSCSI 启动程序清理、文档和操作验证）在迁移后阶段处理，故意超出本节的范围。

.关于此任务
从 ONTAP 9.17.1 开始，ASA r2 系统支持使用 FLI 离线迁移的外部 LUN 的数据迁移。ASA r2 系统与其他 ONTAP 系统（ASA、AFF 和 FAS）在其存储层的实现方面有所不同。在 ASA r2 系统中，创建存储单元（LUN 或命名空间）时会自动创建卷。每个卷仅包含一个存储单元。因此，对于 ASA r2 系统，在创建 LUN 时无需在  `-path` 选项中包含卷名，而应包含存储单元路径。

.步骤
. 将权限级别设置为 advance：
+
[source, cli]
----
set -privilege advanced
----
. 确认导入状态（删除前必须完成或停止）：
+
[source, cli]
----
lun import show -vserver <SVM_NAME> -path /vol/<VOL>/<LUN_NAME>
----
. 如果导入仍处于进行中状态并已完成 % 100：
+
[source, cli]
----
lun import stop -vserver <SVM_NAME> -path /vol/<VOL>/<LUN_NAME>
----
. 删除导入关系（完成）：
+
[source, cli]
----
lun import delete -vserver <SVM_NAME> -path /vol/<VOL>/<LUN_NAME>
----
. 验证关系已删除：
+
[source, cli]
----
lun import show -vserver <SVM_NAME> -path /vol/<VOL>/<LUN_NAME>
----
. 确认导入未处于活动状态（应停止或完成）：
+
[source, cli]
----
lun import show -vserver <SVM_NAME> -path /vol/<VOL>/<LUN_NAME>
----
. 使 LUN 脱机（验证必须脱机完成）：
+
[source, cli]
----
lun offline -vserver <SVM_NAME> -path /vol/<VOL>/<LUN_NAME>
----
. 开始逐块验证（可选，但推荐）：
+
[source, cli]
----
lun import verify start -vserver <SVM_NAME> -path /vol/<VOL>/<LUN_NAME>
----
. 监控验证进度：
+
[source, cli]
----
lun import show -vserver <SVM_NAME> -path /vol/<VOL>/<LUN_NAME>
----
. 停止验证（在使 LUN 联机之前需要）：
+
[source, cli]
----
lun import verify stop -vserver <SVM_NAME> -path /vol/<VOL>/<LUN_NAME>
----
. 使 LUN 联机（中断窗口在此处结束）：
+
[source, cli]
----
lun online -vserver <SVM_NAME> -path /vol/<VOL>/<LUN_NAME>
----
