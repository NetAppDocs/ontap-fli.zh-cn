= ONTAP 外部 LUN 导入 (FLI) 执行阶段
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
执行阶段是执行迁移的地方。它涵盖了运行离线或在线 Foreign LUN Import 以及将主机访问从外部阵列转移到 ONTAP 阵列所需的端到端任务。



== ONTAP FLI 执行阶段任务

下面是每个组件的 ONTAP FLI 执行任务。

[cols="1,2"]
|===
| 组件 | 任务 


| 主机  a| 
* 查看主机事件和系统日志，尽早发现问题，并在任何重大重新配置之前降低风险。
* 根据需要重新启动主机，以确保在早期切换或修复之前保持稳定。
* 切换后，验证主机访问。所有预期的设备都存在，多路径运行正常，应用程序和文件系统正常运行。




| iSCSI FLI 和网络  a| 
* 确认 IC LIF 可到达外部阵列目标门户，并确保在整个导入过程中后端连接保持稳定。
* 验证 iSCSI 会话保持正常，因为它们驱动外来 LUN 发现和路径可用性。




| 网络结构  a| 
* 请确保已设置所需的分区和屏蔽，以便 ONTAP 启动器端口可以访问外部阵列 LUN 以进行发现和导入。
* 切换后，根据需要更新分区，以便主机访问目标 ONTAP LUN 并准备稍后的清理。




| NetApp 存储阵列（目标）  a| 
* 在创建导入作业之前，确认在 ONTAP 中发现了外部 LUN。
* 创建导入关系，并在发现确认后启动导入作业。
* 监控导入进度并处理异常。根据需要使用暂停和恢复来规范复制流，特别是在主机延迟增加的情况下。
* 转换处理：
+
** 离线 FLI 切换在导入完成后发生。
** 在线 FLI 主机切换提前发生，而导入继续在后台执行。






| 外部存储阵列（源）  a| 
* 确保正确显示用于迁移的源 LUN 以供 ONTAP 发现和导入（FC 掩蔽或 iSCSI 目标可见性取决于传输）。
* 请确保在导入过程中未通过主机访问源 LUN。这对于确保写入操作不会从多个来源发生至关重要。
* 在线切换期间，根据需要协调取消映射和重新映射操作，以防止双重呈现并确保主机干净地过渡到 ONTAP。


|===
对于 FC 和 iSCSI 后端 FLI，通过确保 FC/iSCSI 会话健康和到外部 LUN 的完整路径冗余，执行成功在很大程度上取决于保持稳定的后端连接。由于主机 I/O（在线）和导入副本可以竞争相同的后端资源，因此如果需要保护主机延迟，可以在操作上限制导入（例如，使用暂停和恢复），并且通常在非高峰时段安排迁移。
