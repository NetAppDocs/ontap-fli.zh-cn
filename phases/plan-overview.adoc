= ONTAP 外部 LUN 导入 (FLI) 计划和准备阶段
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
计划和准备阶段将发现和分析输出转换为可执行迁移计划，并确认环境已准备好迁移窗口。大多数项目工作通常都放在这里，因为在这里，您可以完成在线或离线切换方法，验证端到端连接，并在迁移到生产环境之前通过一个或多个测试迁移来证明工作流。

对于 iSCSI FLI，此阶段还必须验证 iSCSI 后端设计约束。ONTAP iSCSI 软件启动器使用 IC LIF，需要明确的目标门户 IP 和目标 IQN，无发现会话，并设计为在定义的规模限制（每个节点导入、每个 LIF/节点会话、每个 LIF 启动器、最大导入大小）内运行。



== ONTAP FLI 计划和准备阶段结果

* 使用主机差距分析完成补救计划，包括迁移前所做的更改，包括在切换窗口期间所做的更改，例如 MPIO 更改。
* 构建源到目标 LUN 映射，源 LUN 标识符和序列号到目标 ONTAP LUN 名称和路径，包括回滚映射。
* 使用 IC LIF 可达性和完整网状会话拓扑验证从 ONTAP 到外部阵列的后端连接，其中每个 HA 节点建立到每个目标门户的会话。这是可靠的外部 LUN 发现的先决条件。
* 执行一个或多个测试迁移，以确认配置正确性并收集有助于估计生产持续时间和人员配备工作的吞吐量示例。




== ONTAP FLI 计划和准备阶段任务

以下是每个组件的 ONTAP FLI 计划和准备任务。

[cols="1,2"]
|===
| 组件 | 任务 


| 主机  a| 
* 使用差距分析最终确定主机修复计划，包括更改应在迁移前、切换期间还是迁移后进行。
* 根据停机容忍度和工作负载临界度，选择在线或离线的迁移模式；相应地调整补救时间。
* 验证并记录迁移窗口期间要使用的预期使用情形、标准 LUN、群集磁盘、RDM/VMFS 数据存储区等的主机端重新发现和修正步骤。




| 结构 / 网络  a| 
* FC 环境：备份现有区域集并计划切换和迁移后清理所需的区域更改。
* iSCSI 后端环境：验证 ONTAP IC LIF 和外部阵列目标门户 IP 之间的端到端 IP 可达性（路由/VLAN/防火墙，包括 TCP/3260），并确保冗余。
* 当现有的 IC LIF 已在另一个子网上使用时，请勿在不同的子网上添加新的 IC LIF，以达到外部目标（视为不受支持的规划）。




| NetApp 存储阵列（目标 ONTAP）  a| 
* 确认 ONTAP 集群对 iSCSI FLI（支持的版本/ECV）的准备情况，以及仅迁移的 iSCSI 后端配置将在迁移后删除（对升级/恢复准备很重要）。
* 在拥有 LUN 的节点和 HA 合作伙伴上识别并确认 IC LIF，并规划弹性（通常在不同端口上每个节点两个 IC LIF）。
* 创建软件 iSCSI 启动器（每个 IC LIF 一个启动器）并建立完整的网状会话（每个 HA 节点对应每个外部目标门户和 IQN）并验证会话是否正常。
* 使用 storage disk show 命令在 ONTAP 中发现外部 LUN，验证路径运行状况和基本属性（包括 512B 要求），然后将外部 LUN 标记为外部。
* 使用 -foreign-disk 参数设置具有大小奇偶校验的测试目标 LUN，使用计划前端协议（FC 或 iSCSI）映射它，并执行至少一个测试迁移以验证端到端工作流和捕获吞吐量。
* 确保计划遵循 iSCSI FLI 约束（无发现会话、无 MCS、仅 ERL0、导入和会话限制以及最大导入大小）。




| 外部存储阵列（源）  a| 
* 收集目标门户 IP 和目标 IQN，显式创建会话，不使用发现会话。
* 为 ONTAP 启动器创建供应商端访问对象（主机/主机组等效物），并将小型测试 LUN（通常约 1 GB）仅映射到 ONTAP 启动器以进行鉴定和测试。
* 验证外部目标是否看到 ONTAP 启动器登录和会话，以及是否映射了要导入的正确 LUN。
* 确保生产主机在切换期间不会同时呈现相同的源 LUN，以避免双重呈现。
* 在生产导入之前，使用支持的鉴定工具和方法完成外部阵列鉴定。


|===
