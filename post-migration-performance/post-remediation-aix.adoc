= 执行 ONTAP FLI AIX 迁移后主机修复
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
迁移后，AIX 主机需要采取补救措施，以确保新导入的 LUN 得到正确识别并正常运行。



== 迁移后 - 离线 FLI

对于脱机 FLI，删除 LUN 导入并确认主机启动器已连接后，使 LUN 联机。

| 步骤 1：重新扫描所有映射 AIX 主机上的存储 | 步骤 1：重新扫描所有映射的 AIX 主机上的存储。 | 步骤 1：重新扫描所有映射 AIX 主机上的存储 | 步骤 1：重新扫描所有映射 AIX 主机上的存储 | 步骤 1：打开服务器电源 


| 步骤 2：识别多路径设备 | 步骤 2：识别多路径设备 | 步骤 2：识别多路径设备 | 步骤 2：识别多路径设备 | 步骤 2：验证 IPL（Initial Program Load）/SMS 菜单中的启动设备列表将列出引用来自新存储设备的设备列表的新 LUN。将引导顺序设置为新的设备列表路径

|  |  | 第 5 步：列出 VG 上存在的 LV | 第 5 步：验证集群状态和正在使用的应用程序。 |

NOTE: 有关最新和更新的命令列表，请参阅操作系统供应商文档。

.步骤
. 分离后重新扫描以确认清洁状态。
+
[source, cli]
----
cfgmgr
----
. 识别存储设备。
+
[source, cli]
----
lsdev -Cc disk
sanlun lun show all
sanlun lun show all -pv
lsmpio -l hdiskx
----
. 验证设备是否已连接。
+
[source, cli]
----
lspv |grep hdiskx
----
. 标识卷组和文件系统。
+
[source, cli]
----
lspv
lsvg <vg_name>
lsvg -l <vg_name>
lsvg -p <vs_name>
----
. Varyonvg 和挂载文件系统。
+
[source, cli]
----
varyonvg <vg_name>
mount /mount_point
df -g
----




== 迁移后 - 在线 FLI

删除导入关系后，确认新 LUN 在启用 UNMAP 的情况下独立运行。删除 LUN 导入后，检查 LUN 是否设置了 `-space-reserve disabled`和 `-space-allocation enabled`。如果没有，请相应修改。

[cols="4"]
|===
| Filesystem | 块设备 | LVM（文件系统） | Sanboot 


| 步骤 1：重新扫描所有映射 AIX 主机上的存储 | 步骤 1：重新扫描所有映射的 AIX 主机上的存储。 | 步骤 1：重新扫描所有映射 AIX 主机上的存储 | 步骤 1：重新扫描所有映射 AIX 主机上的存储 


|  |  | 如果设备为精简配置，则会自动启用 SCSI UNMAP。 | 如果设备为精简配置，则会自动启用 SCSI UNMAP。 
|===
.步骤
. 分离后重新扫描以确认清洁状态。
+
[source, cli]
----
cfgmgr
----
. 识别存储设备。
+
[source, cli]
----
lsdev -Cc disk
sanlun lun show all
sanlun lun show all -pv
lsmpio -l hdiskx
----
. 验证设备是否已连接。
+
[source, cli]
----

----
. 在设备上验证 UNMAP：
+
[source, cli]
----
chfs -a reclaim=normal /mount_point
lvmstat -v vg_name -r
----
